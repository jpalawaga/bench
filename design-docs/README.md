# Benchpress Design Docs

Living specification for the Benchpress workout tracker. Each file covers an independent topic so agents can load only the context they need.

| Document | When to load |
|----------|-------------|
| [product-requirements.md](product-requirements.md) | Understanding what the app does, user-facing behavior, and feature decisions |
| [workout-flow.md](workout-flow.md) | Working on the workout view, block lifecycle, or navigation between sub-views |
| [data-models.md](data-models.md) | Changing the schema, storage layer, or repository interface |
| [architecture.md](architecture.md) | Understanding the tech stack, project structure, state management, or storage abstraction |
| [ux-decisions.md](ux-decisions.md) | Working on UI components, styling, or interaction patterns |
| [pwa-and-deployment.md](pwa-and-deployment.md) | Anything related to offline support, service workers, GitHub Pages, or iOS quirks |
| [open-questions.md](open-questions.md) | Planning future work or picking up unresolved design items |
